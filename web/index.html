Temporal Representation in Agent Systems

How well do LLMs turn a user's time expression — “last week”, “a month ago”, “weekends in March” — into the concrete dates an agent must pass to downstream tools? An experiment and resulting recommendations.

The interactive reports

Graded evaluation 2-accuracy/accuracy

The 100+ question dataset with every answer scored against the key, with errors weighted by severity (off-by-one vs. totally wrong). Grouped by answering method per question category.

Preference grid 1-measurement/preferences

~29 business time phrases asked at 7 different anchor dates to elicit what concrete dates each model chooses in that scenario. A rough measure of model preferences.

Measuring determinism 1-measurement/consistency

How consistently each model answers questions, whether it invents times for no-time questions, and which reading it picks on ambiguous ones.

Multi-step threading 3-tooling/threading

Measuring whether models can keep times accurate across a multi-turn conversation using the IR or ISO representations, plus an evaluation if tools should accept the IR or ISO forms as arguments.

Compound decomposition 3-tooling/decomposition

Measurements of how well multi-part or filtered time questions can be translated by models into concrete dates and passed to tool calls.

Routing discrimination 4-routing/routing

Our measurements recommended an architecture: ISO for easy times, SCATE-lite for hard ones. How well can agents be taught when to use ISO and when to delegate to the resolver?

Reusable artifacts (lift them into your project)

The temporal dataset src/datasets/

139 graded items across 8 query slices — anchors, the hand-verified answer key, custom-preset definitions, and crib sheets. Framework-agnostic JSON export; bring your own model.

ScateLite IR + resolver src/scate-lite/

The compact time-expression grammar, the deterministic IR→ISO resolver, the locked conventions, and the interval/calendar math. Zero framework dependencies — drop into any TypeScript project.

Mastra time scorers src/scorers/

Drop-in createScorer wrappers — exact / distance / IoU / set-F1, the error-severity ladder, and the production guardrail scorer for catching malformed time args at runtime.

Pure scoring functions src/scoring/

The same scoring logic with no framework imports — if you are not on Mastra, grade against the key with these directly.
